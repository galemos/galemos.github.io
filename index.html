O que são tabelas e posicionamentos CSS?

As tabelas são definidas com a tag <table> . Uma tabela é dividida em linhas (com a tag <tr> ) onde tr significa table row, ou linha da tabela, e cada linha é dividida em células de dados (com a tag <td> ). As letras td significam "table data," que é o conteúdo de uma célula de dados. Uma célula de dados pode conter texto, imagens, listas, parágrafos, formulários, filetes horizontias, tabelas, etc.

Veremos como montar tabelas, adicionar bordas e posiciona-las de forma a comportar suas informações de forma organizada.

Tabelas

A função de uma tabela e a de tabular em colunas e linhas informações, uma posição especifica da tabela é denominada célula e é possível colocar qualquer elemento dentro de uma célula tais como imagem, texto, links e mais. Em geral separa-se uma linha e/ou coluna como cabeçalho para a tabela. Para melhor imaginar como é uma tabela, relacione uma tabela com um tabuleiro de xadrez onde cada quadrado é uma célula.

  • Marcação de tabelas:

    Uma tabela deve estar contida dentro desta tag <table> ... </table> para representar a marcação de uma tabela.

  • Elementos th, caption e atributo summary:

    1. Uma tabela HTML tem dois tipos de células:

      • Célular Cabeçalho <th> : Célula que contém informações de cabeçalho, o texto em um cabeçalho é em negrito por padrão.

      • Células Comuns <td> : Célula que contém os dados e informações, o testo em uma célula comum é regular e alinhada a esquerda por padrão.

    2. A tag <caption> define uma legenda para a tabela e deve ser inserida imediatamente após a abertura da tag <table>.

      • Observação: Você só pode especificar uma legenda por tabela.

      • Dica: Por padrão, a legenda da tabela ficará centralizada e acima da tabela, pode-se usar propriedades do CSS como text-align e caption-side para re-alocar e alinhar a legenda.

    3. O atributo summary não é propriamente um elemento como caption, mas um atributo para o elemento table. O conteúdo de summary não é renderizado na tela de navegadores gráficos mas serve como dado auxiliar para entendimento da tabela para usuários de leitores de tela e displays Braille.

      summary deve ser usado para descrever a finalidade primária da tabela e dar uma indicação da sua estrutura geral. A maioria das tecnologias assistivas lê o conteúdo de summary fornecendo logo no início da tabela uma informação que ajudará o usuário a interpretar a tabela. Para tabelas mais complexas o uso de summary é de muita importância.

      • Observação: <table summary="text">.

  • Elementos thead, tbody e tfoot:

    A tag <thead> é usada para agrupar o conteúdo do cabeçalho de uma tabela. É usada jungo com as tags <tbody> e <tfoot>, que especificam cada parte da tabela (header, body e footer = cabeçalho, corpo e rodapé).

    Os navegadores podem usar esses elementos para ativar a rolagem do corpo da tabela, independentemente do cabeçalho e do rodapé. Além disso, ao imprimir uma tabela grande que abrange várias páginas, esses elementos podem permitir que o cabeçalho e o rodapé da tabela sejam impressos na parte superior e inferior de cada página.

    • Nota: Todas as tags devem ter pelo menos um elemento <tr>.

    • Dica: Apesar deste elemento não afetar o layout da tabela, podem ser usado como elemento para estilização com CSS.

    Modo de uso:

    • A tag <thead> deve ser usada no seguinte contexto: Como um filho de um elemento <table> , depois de qualquer elemento <caption>, <colgroup> , e antes de qualquer <tbody>, <tfoot>, e <tr>.

    • A tag <tbody> deve ser usada no seguinte contexto: Como um filho de um elemento <table>, depois de qualquer elemento <caption>, <colgroup> e <thead>, antes do elemento <tfoot>.

    • A tag <tfoot> deve ser usada no seguinte contexto: Como um filho de um elemento <table>, depois de qualquer elemento <caption>, <colgroup>, <thead> e <tbody>.

  • Atributos id e header:

    1. O atributo 'id' de uma tabela adiciona um identificador, que deve ser único por elemento, para maior identificação do elemento e para que possa ser usado como seletor via CSS. Exemplo: <th id="exemplo">.

    2. HTML 4 introduziu o atributo 'headers' para as células de tabelas <td>. Este atributo é usado em conjunto com o atributo id na célula de cabeçalho <th> com a finalidade de associar uma ou várias células ao seus respectivos cabeçalhos. Exemplo <th header="exemplo">.

  • Elementos colgroup , col e atributo scope:

    1. A tag <colgroup> especifica um grupo de uma ou mais colunas em uma tabela para formatação. É útil para aplicar estilos a colunas inteiras, em vez de repetir os estilos de cada célula, para cada linha.

      • Nota: A tag <colgroup> deve ser um filho de um elemento <table>, depois de qualquer elemento <caption> e antes de qualquer elemento <thead>, <tbody>, <tfoot> e <tr>.

      • Dica: Para definir propriedades diferentes para uma coluna dentro de um <colgroup>, use a tag dentro da tag <colgroup>.

    2. A tag <col> especifica as propriedades da coluna para cada coluna dentro de um elemento <colgroup>. A tag <col> é útil para aplicar estilos a colunas inteiras, em vez de repetir os estilos de cada célula, para cada linha.

    3. O atributo scope define uma maneira de associar células de cabeçalho e células de dados em uma tabela, identifica se uma célula é um cabeçalho para uma coluna, linha ou grupo de colunas ou linhas.

      • Nota:Atributo scope não é suportado no HTML5

      • Exemplo:<td scope="col|row|colgroup|rowgroup">.

Dicas para marcar tabelas

  • Exemplo:<td colspan="2">

  • Exemplo:<td rowspan="3">

Estilização de tabelas

Agora vamos aplicar alguns estilos na tabela para que ela seja melhor visualizada. Levando em consideração que as melhores práticas removeram o suporte aos atributos no HTML5, usarei CSS na tabela da sessão anterior.

  • Bordas:

    O atributo border especifica se uma borda deve ser exibida ao redor das células da tabela ou não.

    • Exemplo: <table border="1|0">.

  • Tipografia e posicionamentos:

    1. As propriedades de font-family deve conter vários nomes de fonte como um sistema de "fallback". Se o navegador não suportar a primeira fonte, ele tentará a próxima fonte e assim por diante.

      Comece com a fonte desejada e termine com uma família genérica, para permitir que o navegador escolha uma fonte semelhante na família genérica, se nenhuma outra fonte estiver disponível.

      • Exemplo: 

        
table, th, td {
    font-family: "Times New Roman", Times, serif;
}

    2. A propriedade align define o alinhamento horizontal da tabela(left, center ou righ = esquerdo, centro ou direita) do conteúdo em <th> ou <td>. Por padrão, o conteúdo dos elementos <th> é alinhado ao centro e o conteúdo dos elementos <td> é alinhado à esquerda e do elemento <table> a esquerda.

      • Exemplo:<table align="left|center|right">

      A propriedade valign define o alinhamento vertical do conteúdo da célula(top, middle ou bottom = topo, centralizado ou base) em <th> ou <td>. Por padrão, o conteúdo dos elementos são middle(centralizado).

      • Exemplo:<tr align="left|center|right" valign="top|middle|bottom">

  • Cores de fundo:

    É possível alterar a cor de fundo da tabela com o atributo bgcolor.

    • Exemplo:<table align="left|center|right" valign="top|middle|bottom">

  • Imagens de fundo:

    É possível alterar o fundo da tabela e colocar uma imgaem com o atributo background.

    • Exemplo:<table background="imagem.jpeg">

Esquemas de posicionamento CSS

O posicionamento dos elementos é uma parte importante da criação dos layouts. Assim, precisamos conhecer e entender as possibilidades que temos dentro da propriedade position, isto é, as opções fixed, relative e absolute. A propriedade position também aceita o valor static, que representa o valor padrão, mas ele não indica posicionamento algum, ou seja, é o valor para "nenhum posicionamento específico" do elemento.

  • Esquema Relativo:

    A primeira opção de posicionamento para um elemento é a relative. Como o nome sugere, ela especifica uma posição relativa do elemento em relação a algo: o elemento pai.

    A posição é definida através de quatro outras propriedades: top, bottom, left e right, que indicam a distância com relação ao topo, base, esquerda e direita, respectivamente, tomando como referência o posicionamento do elemento pai.

  • Esquema Absoluto:

    O valor absolute é o mais complicado. Este valor se comporta como o fixed, porém tendo como referência a posição do elemento relativo mais próximo de onde está contido, ao invés do viewport. Se um elemento position:absolute não possuir elementos ancestrais posicionados relativamente, ele utilizará o body como referência.

  • Esquema Flutuante:

    Outra propriedade CSS utilizada para o layout é o float. O Float serve pra quebrar o texto ao redor de um elemento.

Posicionamento com z-index

A propriedade z-index especifica a ordem da pilha de um elemento.

Um elemento com maior ordem de empilhamento sempre está na frente de um elemento com uma ordem de pilha menor.

  • Nota:O z-index só funciona nos elementos posicionados (position: absolute, position: relative, position: fixed ou position: sticky).

  • Sintaxe: z-index: auto|number|initial|inherit ).

HTML


<h1>The z-index Property</h1>
<img class="img-zindex" src="bg.jpg" width="100" height="140">    
<p>Porque a imagem tem um z-index de -1, ela será colocada atrás do título.</p>

CSS


.img-zindex {
    position: absolute;
    left: 0px;
    top: 0px;
    z-index: -1;
}
